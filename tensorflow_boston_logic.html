TensorFlow波士顿房价预测代码逻辑分析

核心目标：使用TensorFlow构建神经网络，对波士顿房价数据集进行回归预测

📊 整体流程图

🏗️ 神经网络结构

13 → 16 → 1 全连接网络

输入层 (13个特征) → 隐藏层 (16个神经元 + ReLU) → 输出层 (1个神经元 + 线性)

激活函数：隐藏层使用ReLU，输出层使用线性激活（回归任务）

参数总量：13×16 + 16×1 + 16 + 1 = 241个参数

🔄 详细执行流程

1. 数据准备阶段

数据加载：pandas读取housing.csv，无表头，空格分隔
数据形状：506样本 × 13特征 + 1目标变量
特征归一化：MinMaxScaler将特征缩放到[0,1]区间
数据集分割：75%训练集，25%测试集，随机种子42保证可重现

2. 模型构建阶段

API选择：TensorFlow Keras Sequential API
网络架构：tf.keras.layers.Dense(16, activation='relu', input_shape=(13,))
输出层：tf.keras.layers.Dense(1) - 线性激活用于回归
编译配置：Adam优化器(学习率0.01) + MSE损失 + MAE指标

3. 训练阶段

训练参数：300 epochs，batch_size=32
验证策略：训练集中20%作为验证集
监控指标：训练损失、验证损失、MAE
输出模式：verbose=1显示详细训练过程

4. 评估阶段

测试预测：model.predict(test_x)生成预测结果
性能指标：计算测试集MSE和MAE
样本分析：展示前10个测试样本的预测详情

5. 可视化阶段

损失曲线：训练损失 vs 验证损失随epoch变化
预测散点图：真实房价 vs 预测房价，理想对角线
字体配置：中文字体设置解决显示乱码问题

📈 关键技术要点

🎯 性能预期

典型结果：
• 测试集MSE: ~15-25 (取决于随机初始化)
• 测试集MAE: ~2.5-4.0 (平均预测误差)
• 收敛速度: 通常在50-100 epochs内达到较好性能
• 过拟合控制: 通过验证集监控损失变化
